> 它规定了项目是什么、用什么技术、怎么协作。修改它即可改变 Claude Code 的默认行为。

---

## 项目简介

大学生 AI 科研助手（网页版）。帮助大学生（尤其理工科）更高效地读文献、
处理实验数据、做学术输出。详细规划见 `PROJECT_PLAN.md`。

**当前目标（MVP）**：上传 PDF 论文 → AI 生成结构化总结 → 与论文对话。
在 MVP 跑通前，不要开发其他功能。

---

## 技术栈（请始终保持一致，不要随意更换）

- 框架：Next.js（App Router）+ TypeScript
- 样式：Tailwind CSS
- UI 组件：shadcn/ui
- 部署：Vercel
- 数据库 / 登录 / 文件存储：Supabase
- AI：Claude API（Anthropic）。默认用 Claude Sonnet 4.6 处理总结与对话。
- PDF 解析：unpdf（或 pdf-parse）
- 图表：Recharts（后续阶段才用到）

## 与用户（项目所有者）的协作规则 ★最重要★

用户**没有编程基础**。务必遵守：

1. **一次只做一个小功能。** 做完后停下来，告诉用户怎么测试，等用户确认通过，再做下一个。
   不要一口气实现多个功能。
2. **遇到多种实现方案时，先列出来让用户选**，并说明每种的优缺点，不要自己直接决定。
3. **写完代码后自己运行验证**（如 `npm run dev`），把任何报错贴出来并修复，
   不要假设代码能跑。
4. **每一步都用通俗中文解释**你在做什么、为什么这么做，避免堆术语。
5. 不确定需求时，**先提问，不要猜**。

## 安全与密钥规则 ★绝不违反★

- 所有密钥（Claude API key、Supabase key 等）只能放在 `.env.local`。
- **绝不**把密钥写死在代码里。
- **绝不**把 `.env.local` 提交到 Git（确保它在 `.gitignore` 里）。
- API 调用只能在后端（`app/api/` 里的接口）进行，**绝不**在前端浏览器里直接调用 AI API
  （否则密钥会泄露）。
- 提供 `.env.example` 作为模板（只写变量名，不写真实值）。

---

## 编码约定

- 用 TypeScript，尽量写清类型。
- 组件文件放 `components/`，工具函数放 `lib/`，类型放 `types/`。
- 把对 Claude API 的调用封装在 `lib/claude.ts`，对 PDF 的处理封装在 `lib/pdf.ts`，
  方便复用和后续修改。
- 界面文案用中文。
- 错误处理：调用 AI 或解析文件失败时，要给用户友好的中文提示，不要让页面崩溃。

## ⚠️ 重要技术红线

实验图像（如 SEM）的**定量**测量（精确粒径、分布等）**不能只靠 AI 视觉模型**，
必须用图像处理库（OpenCV / scikit-image）真实计算。AI 只负责"解读和写结论"。
**绝不让 AI 编造测量数字。** 这部分属于后续阶段，MVP 不涉及。
