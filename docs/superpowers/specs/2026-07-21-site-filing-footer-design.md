# 网站备案信息底栏设计

## 目标

在网站所有页面稳定展示正式 ICP 备案与公安备案信息，同时修复根布局中因重复插入备案代码造成的无效 HTML 结构。

## 备案信息

- ICP 备案：`津ICP备2026007356号`，链接至 `https://beian.miit.gov.cn/`
- 公安备案：`冀公网安备13028302000277号`，链接至 `https://beian.mps.gov.cn/`

两个链接均在新标签页打开，并使用 `rel="noopener noreferrer"`。

## 展示方案

备案信息只保留一份，作为全站公共底栏放在根布局中，不再在侧边栏重复展示。底栏使用低对比度小字号，与现有界面保持协调；桌面端横向排列，窄屏自动换行。公安备案项包含公安备案图标和备案号。

底栏参与正常页面布局，不使用会遮挡内容的固定定位。页面主体采用纵向弹性布局，使内容不足一屏时底栏贴近视口底部，内容较长时自然位于文档末尾。

## 结构调整

- 根布局只保留一个 `<body>` 和一个 `<AppShell>`。
- 页面内容、反馈组件、登录成功提示和通知组件保持现有功能。
- 新增一个职责单一的备案底栏组件，由根布局统一渲染。
- 删除侧边栏中的重复备案区域。

## 响应式与可访问性

- 链接具备清晰的悬停和键盘焦点状态。
- 小屏幕允许两项备案信息居中换行，不产生横向滚动。
- 公安备案图标提供有意义的替代文本；装饰分隔符对辅助技术隐藏。

## 验证

- 运行 ESLint 与 TypeScript/Next.js 构建检查。
- 检查生成结构中没有嵌套或重复 `<body>`。
- 检查两条备案号文字及目标链接准确。
- 检查桌面和窄屏下底栏不遮挡主要内容或浮动控件。

## 范围

本次只处理备案展示、根布局结构以及侧边栏重复内容，不重构其他页面或修改现有业务功能。
